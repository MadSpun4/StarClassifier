# Классификатор светимости звёзд

Этот репозиторий содержит реализацию системы, основанной на знаниях, для предметной области «Классификация светимости звёзд». Приложение разрабатывалось как учебный проект по задаче классификации звезды по её астрофизическим характеристикам.

В системе сочетаются два подхода:

- экспертный алгоритм на базе знаний выполняет основную проверку и формирует объяснение результата;
- ML-модель вычисляет вероятностные оценки по классам светимости и используется как дополнительный компонент гибридной классификации.

## Постановка задачи

Объектом классификации является отдельная звезда. Результатом работы системы является определение её класса светимости.

При решении задачи используются следующие свойства звезды:

- эффективная температура;
- поверхностная гравитация;
- абсолютная светимость;
- радиус звезды.

В системе рассматриваются следующие классы светимости:

- I — сверхгиганты и гипергиганты;
- II — яркие гиганты;
- III — гиганты;
- IV — субгиганты;
- V — звёзды главной последовательности;
- VI — субкарлики;
- VII — белые карлики.

## Что реализовано в системе

В приложении выделены основные подсистемы, предусмотренные в документации:

- редактор знаний;
- система ввода исходных данных;
- решатель задач;
- подсистема вывода результата и объяснений.

Редактор знаний позволяет формировать и изменять базу знаний: добавлять классы светимости, свойства звезды, задавать возможные диапазоны значений, описания свойств для классов и конкретные диапазоны значений для каждого класса.

Система ввода исходных данных используется для ввода параметров наблюдаемой звезды. После этого решатель задач определяет подходящий класс светимости, а подсистема вывода показывает результат и объясняет, почему другие классы были опровергнуты.

## Как работает классификация

Основной метод решения задачи — метод опровержения гипотез. Для каждого класса светимости система проверяет гипотезу о том, что исследуемая звезда принадлежит этому классу. Если хотя бы одно значение не согласуется с диапазоном, заданным в базе знаний, класс опровергается.

Если в результате проверки остаётся один подходящий класс, он считается результатом экспертного вывода. Если остаётся несколько не опровергнутых классов, ML-модель используется для их вероятностного ранжирования. В интерфейсе пользователю также выводятся вероятностные оценки ML-модели вместе с результатом классификации.

Таким образом, в проекте реализован именно гибридный подход: экспертный алгоритм и ML-модель используются совместно, но приоритет при принятии решения сохраняется за логикой базы знаний.

## Особенности реализации

- пользовательский интерфейс реализован на `tkinter`;
- база знаний хранится в формате `JSON`;
- для вычислительной части используются `NumPy` и `pandas`;
- нейросетевая модель реализована через `scikit-learn` (`MLPClassifier`);
- сохранение обученной модели выполняется с помощью `joblib`.

Обучающая выборка в проекте формируется синтетически на основе базы знаний. Для каждого класса светимости используются заданные интервалы значений по четырём свойствам, после чего на этих данных обучается вспомогательная ML-модель.

## Что умеет приложение

- редактировать базу знаний через графический интерфейс;
- проверять полноту и корректность базы знаний;
- классифицировать звезду по полному набору параметров;
- работать с неполным набором входных данных;
- выводить объяснения причин опровержения классов;
- показывать вероятностные оценки ML-модели;
- переобучать модель после изменения базы знаний.

## Запуск проекта

### Запуск через Python

Если запускать проект из исходников, достаточно установить зависимости и запустить `main.py`:

```powershell
pip install -r requirements.txt
python main.py
```

### Запуск через `run_ui.bat`

В проекте есть батник:

```powershell
run_ui.bat
```

Он автоматически:

- находит установленный Python;
- создаёт локальное виртуальное окружение `.venv`, если его ещё нет;
- устанавливает зависимости из `requirements.txt`, если это необходимо;
- после этого запускает приложение.

Для этого варианта на компьютере должен быть установлен Python 3.10 или новее.

### Windows-сборка

Для проекта подготовлена сборка через `PyInstaller`. После сборки приложение запускается из папки:

```text
dist\StarClassifier
```

Основной исполняемый файл:

```text
dist\StarClassifier\StarClassifier.exe
```

Важно: для работы нужно хранить всю папку `dist\StarClassifier`, а не только один `.exe`, потому что рядом лежат служебные файлы сборки.

## Переобучение модели

Если база знаний была изменена, ML-модель можно переобучить отдельно:

```powershell
python train_model.py
```

Перед обучением система проверяет корректность базы знаний. Если в ней есть противоречия или незаполненные значения, обучение не начнётся.

## Сборка `.exe`

Установка зависимостей для сборки:

```powershell
python -m pip install -r requirements-build.txt
```

Сборка:

```powershell
.\build_exe.ps1
```

## Структура проекта

- `main.py` — точка входа в приложение;
- `train_model.py` — скрипт переобучения модели;
- `run_ui.bat` — запуск через Python с автоматической подготовкой окружения;
- `build_exe.ps1` — скрипт сборки Windows-версии;
- `StarClassifier.spec` — конфигурация `PyInstaller`;
- `requirements.txt` — зависимости для запуска проекта;
- `requirements-build.txt` — зависимости для сборки;
- `data/knowledge_base.default.json` — эталонная база знаний;
- `data/knowledge_base.json` — рабочая база знаний;
- `models/star_luminosity_mlp.joblib` — обученная модель;
- `models/star_luminosity_mlp.meta.json` — метаданные модели;
- `star_classifier/` — исходный код приложения.

## Коротко о проекте

Это реализация системы, основанной на знаниях, для классификации светимости звёзд. В проекте сделан упор на экспертный вывод, редактируемую базу знаний, объяснение результата и гибридную схему работы с дополнительной нейросетевой оценкой.
